## Make a Lookup Request

Use `GET /lookup` with a US phone number. Send your provided API key in the `X-API-Key` header.

```bash
curl "https://api.callheater.com/lookup" \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "X-Phone: 4155550123" \
  -H "X-Lookup-Mode: Basic" \
  -H "X-Display: json" \
  -H "X-Cache: true"
```

The default response is JSON. Set `X-Display: pretty` when you want a plain text response for humans.

```bash
curl "https://api.callheater.com/lookup" \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "X-Phone: 4155550123" \
  -H "X-Display: pretty"
```

## Choose a Lookup Mode

Use `X-Lookup-Mode: Basic` for standard caller context. Use `X-Lookup-Mode: DeepResearch` when deeper enrichment is available for the number.

```bash
curl "https://api.callheater.com/lookup" \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "X-Phone: 4155550123" \
  -H "X-Lookup-Mode: DeepResearch"
```

## Common Options

Use these headers to control the lookup response:

- `X-Phone`: Required phone number to look up.
- `X-Lookup-Mode`: `Basic` or `DeepResearch`.
- `X-Display`: `json` or `pretty`.
- `X-Cache`: Set to `false` to force a fresh lookup.
- `X-RealEstate`: Set to `true` to include real estate market info in pretty responses.
- `X-Debug`: Set to `true` to include source and timing details for troubleshooting.

The `sports` option is still available as a query parameter when you want area-code sports team data.

## Response

JSON responses include lookup metadata, phone details, carrier and line type signals when available, caller identity signals when available, and billing metadata for the request.

<Note>
  Lookup data is for informational purposes only. Do not use it as the sole basis for personal, legal, financial, or safety decisions.
</Note>
